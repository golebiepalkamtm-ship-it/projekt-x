# Discovery Specification

Discovery accepts user-selected filters and approved contextual signals to return eligible public profiles. Results distinguish direct matches, sponsored placements if any, and recommendations. The product must explain controllable ranking factors in plain language.

Hard safety, eligibility, visibility, and geographic policy constraints run before ranking. Ranking optimisation may not override an enforcement decision. A user can clear personalization, understand why a result appears, and report inappropriate output.

Metrics include successful discovery, query latency, zero-result rate, filter usefulness, reports, and fairness/safety review—not only engagement.
