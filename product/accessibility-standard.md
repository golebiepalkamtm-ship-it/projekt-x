# Accessibility Standard

Core journeys target WCAG 2.2 AA. Interfaces support keyboard-only use, meaningful focus order, visible focus, screen-reader names, semantic headings, sufficient contrast, reduced motion, text scaling, and error recovery without relying on colour alone.

Accessibility acceptance criteria are written with each journey. A component cannot be considered reusable until its keyboard, focus, semantic, loading, error, and responsive behaviours are specified and tested.
