# Model Card Template

## Identity

Name, owner, version, provider, release date, approval status.

## Intended and prohibited use

State user value, decision boundary, affected populations, excluded use, and required human review.

## Data and evaluation

List data categories, lawful basis/approval, quality limits, offline metrics, safety tests, subgroup analysis, and known failure modes.

## Operations

Define logging minimisation, monitoring metrics, thresholds, incident owner, rollback, and review cadence.
