# AI Evaluation Standard

Evaluation begins with an explicit intended outcome and compares the system to a non-AI baseline. Test sets include realistic edge cases, adversarial input, protected-content handling, uncertainty, language variation, and known vulnerable scenarios.

Report accuracy, calibration, false-positive and false-negative cost, subgroup performance where lawful and appropriate, operator reliance risk, latency, and privacy leakage. A model release needs acceptance thresholds and a stop condition; post-release monitoring uses the same definitions.
