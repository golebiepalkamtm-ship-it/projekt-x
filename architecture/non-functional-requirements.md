# Non-functional Requirements

| Area | Baseline requirement | Evidence |
|---|---|---|
| Availability | Critical reads and safety workflows have defined SLOs and graceful degradation. | dashboards and exercises |
| Privacy | Personal data has purpose, owner, retention and deletion path. | data inventory |
| Security | Sensitive actions are authenticated, authorised, rate-limited and audited. | threat model and tests |
| Accessibility | Core flows meet WCAG 2.2 AA. | automated and manual review |
| Performance | User-facing interaction budgets are defined per journey. | synthetic and RUM metrics |
| Recoverability | Data and service recovery objectives are approved before launch. | restore drills |
| Observability | Every critical flow has metrics, traces, logs and owner alerts. | runbook exercise |
