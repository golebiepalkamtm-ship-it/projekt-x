# ADR-0001: Operate ESEP through a documentation-first Project OS

## Context

ESEP is intended to grow across many domains. Conversation-only decisions are hard to discover, verify, and update.

## Decision

The repository is the canonical source for approved product, architecture, policy, and implementation contracts. Material changes require linked documentation and a Project Memory entry.

## Consequences

Delivery starts slower, but review, onboarding, auditing, and change impact become tractable. Documents must be maintained as living artifacts.
